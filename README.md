# Neuro Syntax IDE 架构规划总览

**Neuro Syntax IDE** 是一款为下一代开发者构建的 **AI 原生桌面端集成开发环境**。它摒弃了传统的“工具作为被动编辑器”的定位，引入了系统级权限、原生终端集成以及拥有自主调度能力的人工智能 (PM Agent)，主动参与从需求分析、功能拆解到编码验收的全生命周期。

通过采用 **React 18 + Tauri V2 (Rust)** 混合架构设计，本项目以极低的资源占用获得了媲美原生应用程序的系统级交互性能（真正的 Pty 虚拟终端、底层文件监听、硬件资源探针采集）。

这套文档集合涵盖了项目的整体架构思潮以及可并行开发的详细实施步骤：

* `tauri_architecture_plan.md` - 全局系统架构与框架概览。

* `tauri_development_modules.md` - 并行式模块拆解策略图。

* `module_1_core_workspace.md` / `module_2_task_data_layer.md` ... 等 5 份细分研发卷宗。

## 🚀 核心革新：文档即数据库 (FS-as-Database)

本项目在系统持久化层面上没有采用常见的 SQLite 或 ORM 框架，而是推行了一种极客、纯粹、且对于 AI 及代码开发者来说最为友好的架构——**文档驱动的本地状态树机制（FS-as-Database）**。

我们将应用的核心业务数据（任务看板、状态列、特征流）直接映射为硬盘上带有明确目录结构的 **YAML 状态机 (**`queue.yaml`**) + Markdown 文件 (**`features/`**)**。IDE 通过 Rust `std::fs` 与 `notify` 监听系统文件夹并直接在前端渲染。

### 采用这种方案的四大颠覆性优势：

#### 1. 🤖 对 AI (PM Agent) 极致友好

对于大模型（LLM）而言，“基于表结构的 SQL 操作”非常复杂且容易出现注入或幻觉；而“阅读和书写代码/文本”则是它们的**绝对专长**。\
Agent 在规划出一个 Feature 的需求大纲后，可以直接在 `features/` 生成一个独立的文件夹，并在其中随意抛入 `plan.md`，追加业务逻辑 `evidence` 图片。**无需修改任何数据库表结构，IDE 面板只需读取树状文件即可重绘看板。** 这种直接的文件投递，是人和 AI 之间无隔阂沟通的最高效媒介。

#### 2. 🌳 Git 天然接管一切，免费获得时间机器

既然所有的数据（任务在哪个阶段、每个任务拆分成了哪五小步、需求的上下文）仅仅是一堆 `Yaml` 字符和 `.md` 文本的组合，这意味着你只需在项目目录下执行 `git commit`。\
**整个项目的代码逻辑 + 看板任务状态 + PM Agent 的思考过程被 100% 绑定在了一个 Git Hash 下！** 你可以使用所有的历史工具（Git Blame, Git Log）看到哪个任务在昨天被挪动过，或者轻松让整个 Kanban 看板状态“回滚”至上周三的发版状态，毫无痛点。

#### 3. 🔓 绝对透明，告别“供应商锁定” (Zero Lock-in)

传统的任务管理软件一旦卸载或服务下线，数据便化为乌有；或者其导出的 CSV 格式混乱不堪。\
在 Neuro Syntax IDE 下，即便你不用这个软件，仅使用 VS Code，依然能通过文本编辑器打开 `feature-workflow/queue.yaml` 进行日常编辑并让项目状态向前推进。你的项目资产、所有拆解过程属于你的本地硬盘，属于你本身。

#### 4. 🔀 高效协同与冲突解决

因为看板流就是个文本队列，所以只要开发者之间推陈（Push \ Pull）分支：

* 小王新建了一个 Feature YAML；

* 小李把上个任务的属性标记为 Completed。 传统的协同依赖云端 Websocket 以及极其复杂的数据库锁；但现在，Git 的 Merge 引擎会自动识别文件差异并将大家的工作流合而为一，完美同步！

## 🛠 未来愿景

在这个文档和系统双轮驱动的环境下，当你在 `TaskBoard` 上拖拽一张任务卡片到 "In Progress" 阶段时，应用并非是在调用某个僵化的后端 SQL API，而是在硬盘里优雅地对 `queue.yaml` 中的文字完成了一次裁剪。与此同时，底层的基于 Rust 的 `monitor watcher` 会立刻在几毫秒内将这丝更改映射至屏幕的每一个神经元上。

这是一个为极客和 AI 共同谱写代码而生的世界。
