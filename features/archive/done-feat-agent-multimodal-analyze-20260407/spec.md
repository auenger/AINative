## Description

利用 LLM 多模态能力分析 `{workspace}/PMFile/` 中的上传文件（文档、图片、音频等），将分析结果以结构化 Markdown 文档形式输出到 `{workspace}/PMDM/` 目录。支持批量分析和单文件分析，分析完成后通知 Agent 对话面板。

## User Value Points

1. **一键分析文件** — 用户选择 PMFile 中的文件，Agent 自动分析内容并生成 MD 文档

2. **多格式内容提取** — docx 文本提取、PDF 内容读取、图片视觉分析（OCR + 场景描述）、音频转文字 + 摘要、md 直接解析

3. **结构化 MD 输出** — 分析结果以统一格式的 MD 文档保存到 PMDM 目录，包含元信息、摘要、关键内容、建议

## Technical Solution

### Architecture

The multimodal analysis pipeline consists of three layers:

1. **Tauri Backend (Rust)**
   - `pmfile_read_content` — Unified file content reader that returns base64 for binary files (images, audio, PDF) and text for documents (docx via zip extraction, markdown, text)
   - `pmfile_analyze` — Full analysis pipeline: reads file → builds multimodal Gemini API request → streams response via `analyze://chunk` events → writes final MD to PMDM directory
   - `pmdm_write` / `pmdm_list` — PMDM directory management (write MD files, list existing analysis results)
   - Analysis prompts are embedded in Rust, tailored per file type (image: visual description + UI analysis; audio: transcription + summary; documents: content extraction + requirements identification)

2. **Frontend Hook (`useMultimodalAnalyze`)**
   - Manages per-file analysis state (pending / analyzing / done / error)
   - Registers Tauri event listeners (`analyze://chunk`, `analyze://complete`, `analyze://error`) for streaming progress
   - Supports single-file and batch analysis
   - Tracks which files have already been analyzed by scanning the PMDM directory
   - Provides `isAnalyzed()` check and overall progress calculation

3. **UI Integration**
   - `FileUploadArea` component extended with analyze props (per-file Sparkles button, batch "Analyze All" bar)
   - Analyzed files show green CheckCircle2 badge
   - Analyzing files show animated Loader2 + progress bar
   - Analysis count displayed (e.g., "2/5 analyzed")

### Data Flow

```
PMFile/{file} → pmfile_read_content → [base64/text] → Gemini Multimodal API → stream → analyze://chunk events → Frontend state → PMDM/{file}-analysis.md
```

### MD Output Template

```markdown
# 文件分析报告

## 元信息
- **文件名**：{filename}
- **文件类型**：{file_type}
- **分析时间**：{timestamp}

## 分析内容

{LLM-generated analysis}

*Generated by Neuro Syntax IDE Multimodal Analyzer*
```

## Acceptance Criteria (Gherkin)

### User Story

作为项目管理者，我希望 Agent 能自动分析我上传的参考文档和图片，并生成结构化的 Markdown 分析报告，以便后续在需求分析中使用。

### Scenarios (Given/When/Then)

#### Scenario 1: 分析 PDF 文档

```gherkin
Given PMFile 目录中有一个 product-requirements.pdf 文件
When 用户在 Agent 面板中点击"分析"按钮并选中该 PDF
Then 系统应提取 PDF 文本内容并通过 LLM 分析
And 分析结果以 MD 格式保存到 {workspace}/PMDM/product-requirements-analysis.md
And MD 文档包含：文件元信息、内容摘要、关键要点、需求建议
```

#### Scenario 2: 分析图片文件

```gherkin
Given PMFile 目录中有一个 wireframe.png 设计稿
When 用户选中该图片进行多模态分析
Then 系统应通过 LLM 视觉能力分析图片内容
And 生成包含 UI 元素描述、布局分析、交互建议的 MD 文档
```

#### Scenario 3: 分析音频文件

```gherkin
Given PMFile 目录中有一个 meeting-recording.wav 文件
When 用户选中该音频进行分析
Then 系统应通过 LLM 音频能力（或先转文字）分析内容
And 生成包含转录摘要、关键决策、待办事项的 MD 文档
```

#### Scenario 4: 批量分析多个文件

```gherkin
Given PMFile 目录中有 3 个文件（docx, pdf, png）
When 用户点击"全部分析"按钮
Then 系统应依次分析所有文件
And 每个文件对应一个 MD 文档输出到 PMDM 目录
And Agent 面板显示分析进度
```
